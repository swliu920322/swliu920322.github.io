前言

由于之前一直是手动打包，比较忙的时候测试还来找你打包，所以想把打包资源共享出来。

  • 第一步，先本地实现命令行打包
  • 第二步，实现桌面端调用
  • 第三步，实现前端页面配置
  • 第四部，暴露公网，让别人远程调用

安卓

  1. 配置gradle全局命令
    • 在.bash_profile里增加
    • ```
      GRADLE_HOME=/Users/liushengwei/.gradle/wrapper/dists/gradle-7.3.3-bin/6a41zxkdtcxs8rphpq6y0069z/gradle-7.3.3
      export GRADLE_HOME
      export PATH=${PATH}:${GRADLE_HOME}/bin
         - source .bash_profile里增加
2. 配置一系列的参数 
   - signing.properties
      STORE_FILE = ./xydti.keystore
      STORE_PASSWORD = 123456
      KEY_ALIAS = xydti
      KEY_PASSWORD = 123456 
          -
3. 执行 gradle assemble

### ios打包

1. 第一步就是配置好配置，使得可以本地打包能启动
2. 第二步看这个[Xcode命令行脚本打包](https://www.jianshu.com/p/e41b7dbc8aca)

### 遇到的问题

- 升级完 gradle 7.3.3 gradle命令行会报
    - ```Android Gradle plugin requires Java 11 to run. You are currently using Java 1.8.```
    - 升级[java11](https://repo.huaweicloud.com/java/jdk/11.0.2+9/jdk-11.0.2_osx-x64_bin.dmg)
    - 不用配置home
    - 在工程的gradle.properties中添加参数
      org.gradle.java.home=/Library/Java/JavaVirtualMachines/jdk-11.0.2.jdk/Contents/Home 
          - 具体看这个帖子(https://blog.csdn.net/just_hu/article/details/125822130)

### 集成fir

- node集成 使用简单，没有cli高效
    - 主要是个人node写得少,先拿到上传token和key和url，然后走上传
    - 一般分两步，上传图标和上传文件
  ```js
    async function getInfoBeforeUpload() {
      const [ios] = await Promise.all([iosInfo(), androidInfo()]);
      const { key, token, upload_url } = ios.binary;
      const file = fs.createReadStream(filePath);
      request(
        {
          method: 'POST',
          url: upload_url,
          headers: { 'Content-Type': 'multipart/form-data' },
          formData: {
            file,
            key,
            token,
            'x:name': '包名',
            'x:version': '1.0.3',
            'x:build': '1',
            'x:release_type': 'Adhoc',
            'x:changelog': '打包备注',
          },
        },
        function (err, res, body) {
          if (err) console.log(err);
          console.log(JSON.parse(body).download_url);
        }
      );
    }
  • fir-cli集成 集成稍微麻烦，效率比较高
    • 安装ruby
    • 切换ruby
      fir publish ${iosPath} -R --specify-icon-file ${iconPath} -c ${changelog} -s ${short} --specify-app-display-name ${appName}  -Q -T ${api_token}
fir publish ${androidPath} -R --specify-icon-file ${iconPath} -c ${changelog} -s ${android_short} --specify-app-display-name ${appName} -Q -T ${api_token}
